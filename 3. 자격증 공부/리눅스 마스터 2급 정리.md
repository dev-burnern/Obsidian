
## 1.1 파일시스템 관련 명령어

파일시스템 관련 명령어는 리눅스 시스템에서 파일과 디렉토리를 관리하는 데 사용되는 기본적인 명령어들입니다[1](https://velog.io/@sionyy/%EB%A6%AC%EB%88%85%EC%8A%A4%EB%A7%88%EC%8A%A4%ED%84%B0-2%EA%B8%89-2%EC%B0%A8%EC%8B%9C%ED%97%98-%EC%9A%94%EC%95%BD). 이러한 명령어들은 리눅스 사용에 있어 가장 기초적이면서도 빈번하게 사용되는 부분입니다[6](https://hongchange.tistory.com/entry/%ED%8C%8C%EC%9D%BC-%EC%8B%9C%EC%8A%A4%ED%85%9C-%EB%AA%85%EB%A0%B9%EC%96%B4-%EC%A0%95%EB%A6%AC).

|---|---|---|

## 권한 관련 명령어

|명령어|설명|사용 예시|
|---|---|---|
|`chmod`|파일/디렉토리 권한 변경|`chmod 755 script.sh`|
|`chown`|파일/디렉토리 소유자 변경|`chown user:group file.txt`|

파일 시스템 관련 명령어들은 리눅스 시스템을 관리하는 데 필수적인 도구입니다[8](https://choraengyi.tistory.com/39). 대부분의 작업은 이러한 기본 명령어들의 조합으로 이루어집니다[6](https://hongchange.tistory.com/entry/%ED%8C%8C%EC%9D%BC-%EC%8B%9C%EC%8A%A4%ED%85%9C-%EB%AA%85%EB%A0%B9%EC%96%B4-%EC%A0%95%EB%A6%AC).

## 1.2 셸(Shell)

셸은 사용자와 리눅스 커널 사이의 인터페이스 역할을 하는 프로그램입니다[9](https://hyess.tistory.com/11). 사용자가 입력한 명령어를 해석하여 커널에 전달하고, 커널의 처리 결과를 사용자에게 보여주는 역할을 합니다[10](https://velog.io/@devmin/%EB%A6%AC%EB%88%85%EC%8A%A4-%EC%89%98-%EA%B8%B0%EB%B3%B8-%EB%AA%85%EB%A0%B9%EC%96%B4Basic-Shell-Commands).

## 주요 셸 종류

|셸|설명|특징|
|---|---|---|
|Bash (Bourne Again Shell)|대부분의 리눅스 배포판 기본 셸|확장된 기능, 명령어 이력, 자동 완성 지원|
|sh (Bourne Shell)|유닉스의 원래 셸|POSIX 호환, 최소 기능만 제공|
|csh (C Shell)|C 언어와 유사한 구문|C 프로그래머들에게 친숙한 구문|
|tcsh|csh의 확장 버전|명령어 이력, 자동 완성 기능 추가|
|zsh|강력한 기능의 고급 셸|다양한 플러그인, 테마 지원|

## 셸 기본 문법과 특수 문자

|문자/기능|설명|예시|
|---|---|---|
|파이프(`\|`)|한 명령의 출력을 다른 명령의 입력으로 전달|`ls -l \| grep "txt"`|
|리다이렉션(`>`, `>>`, `<`)|출력/입력 방향 변경|`ls > file.txt`|
|와일드카드(`*`, `?`)|파일명 패턴 매칭|`ls *.txt`|
|환경변수|셸 환경 설정 값|`echo $HOME`|
|별칭(alias)|명령어 별칭 설정|`alias ll="ls -la"`|

셸 스크립트는 여러 명령어를 순차적으로 실행할 수 있게 해주는 도구로, 시스템 관리 자동화에 매우 유용합니다[11](https://bioinfoblog.tistory.com/11). 반복적인 작업을 효율적으로 처리할 수 있어 시스템 관리자들이 자주 사용합니다[10](https://velog.io/@devmin/%EB%A6%AC%EB%88%85%EC%8A%A4-%EC%89%98-%EA%B8%B0%EB%B3%B8-%EB%AA%85%EB%A0%B9%EC%96%B4Basic-Shell-Commands).

## 1.3 프로세스 관리

프로세스는 실행 중인 프로그램의 인스턴스를 말하며, 리눅스에서 프로세스 관리는 시스템 자원을 효율적으로 사용하는 데 매우 중요합니다[12](https://bookfriend.tistory.com/entry/ps-top-kill-htop-%EC%99%84%EC%A0%84%EC%A0%95%EB%B3%B5-%EB%A6%AC%EB%88%85%EC%8A%A4-%ED%94%84%EB%A1%9C%EC%84%B8%EC%8A%A4-%EA%B4%80%EB%A6%AC-%ED%95%B5%EC%8B%AC-%EC%A0%95%EB%A6%AC). 리눅스는 다중 사용자, 다중 프로세스 운영체제이므로 여러 프로세스를 동시에 실행하고 관리할 수 있습니다[14](https://skstp35.tistory.com/399).

![리눅스마스터 2급 2차 시험 대비 - 프로세스 관리 명령어 정리표](https://pplx-res.cloudinary.com/image/upload/v1748840955/pplx_code_interpreter/c0254fce_cguq1e.jpg)

## 프로세스 상태

|상태|설명|
|---|---|
|Running (R)|실행 중이거나 실행 대기 중|
|Sleeping (S)|인터럽트나 이벤트 대기 중 (인터럽트 가능)|
|Uninterruptible Sleep (D)|디스크 I/O 등으로 인한 대기 상태 (인터럽트 불가능)|
|Stopped (T)|중지된 상태 (SIGSTOP 신호로 인해)|
|Zombie (Z)|종료되었지만 부모 프로세스가 상태를 수집하지 않은 상태|

## 주요 시그널

|시그널|번호|설명|
|---|---|---|

프로세스 관리는 시스템 성능 모니터링과 최적화에 필수적인 부분이며, 특히 서버 환경에서 중요합니다[12](https://bookfriend.tistory.com/entry/ps-top-kill-htop-%EC%99%84%EC%A0%84%EC%A0%95%EB%B3%B5-%EB%A6%AC%EB%88%85%EC%8A%A4-%ED%94%84%EB%A1%9C%EC%84%B8%EC%8A%A4-%EA%B4%80%EB%A6%AC-%ED%95%B5%EC%8B%AC-%EC%A0%95%EB%A6%AC). 리눅스 시스템에서 적절한 프로세스 관리는 시스템의 안정성과 성능을 유지하는 데 도움이 됩니다[14](https://skstp35.tistory.com/399).

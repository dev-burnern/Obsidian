## 파일의 마지막 줄 출력 명령어

## 다음 중 파일의 마지막 (10, 5, 20)줄을 출력하는 명령어는?

✅ 정답: `tail`

- `tail` 명령어는 기본적으로 파일의 마지막 10줄을 출력합니다
- 출력할 줄 수를 지정하려면 `-숫자` 또는 `-n 숫자` 옵션을 사용합니다.
    - 마지막 5줄: `tail -5 파일명` 또는 `tail -n 5 파일명`
    - 마지막 20줄: `tail -20 파일명` 또는 `tail -n 20 파일명`

**예시:**
```Linux
[user1@localhost ~]$ 
tail -7 /etc/services edi_service 34567/udp \# dhanalakshmi.org EDI Service axio-disc 35100/tcp # Axiomatic discovery protocol axio-disc 35100/udp # Axiomatic discovery protocol pmwebapi 44323/tcp # Performance Co-Pilot client HTTP API cloudcheck-ping 45514/udp # ASSIA CloudCheck WiFi Management keepalive cloudcheck 45514/tcp # ASSIA CloudCheck WiFi Management System spremotetablet 46998/tcp # Capture handwritten signatures
```

**추가 기능:** `tail -f` 옵션을 사용하면 파일 내용의 변화를 실시간으로 모니터링할 수 있습니다(로그 파일 추적에 유용)[13](https://joonius.tistory.com/m/37).

## 텍스트 파일 출력 방식

## 텍스트 파일을 (페이지, 전체 단위)로 출력하는 명령어는?

✅ 정답:

- 페이지 단위: `more`, `less`
    
- 전체 단위: `cat`
    

## 페이지 단위 명령어

1. **more**
    
    - 파일 내용을 화면 단위로 출력합니다[3](https://www.tutorialspoint.com/unix_commands/more.htm).
        
    - 스페이스바를 누르면 다음 페이지로 이동합니다.
        
    - 엔터 키를 누르면 한 줄씩 스크롤됩니다.
        
    - 단방향(앞으로만) 스크롤이 가능합니다.
        
    - `q`를 누르면 종료됩니다.
        
2. **less**
    
    - `more`의 확장된 버전으로, 더 많은 기능을 제공합니다[4](https://server-talk.tistory.com/402)[5](http://www.linux-magazine.com/Issues/2023/273/New-Kid-on-the-Block).
        
    - 양방향 스크롤이 가능하여 앞뒤로 이동할 수 있습니다.
        
    - 주요 키:
        
        - `j`: 다음 행으로 스크롤
            
        - `k`: 이전 행으로 스크롤
            
        - 스페이스바 또는 `Ctrl+f`: 다음 화면
            
        - `Ctrl+b`: 이전 화면
            
        - `/검색어`: 문자열 검색
            
        - `q`: 종료
            

## 전체 단위 명령어

- **cat**
    
    - 파일의 내용을 한 번에 모두 출력합니다[3](https://www.tutorialspoint.com/unix_commands/more.htm).
        
    - 스크롤이나 페이지 나누기 없이 모든 내용이 연속적으로 표시됩니다.
        
    - `-n` 옵션을 사용하면 행 번호를 붙여 출력합니다.
        

## 중첩 디렉터리 생성

## (var/www/html) 디렉터리가 존재하지 않을 경우. 상위 디렉터리까지 포함하여 생성하도록 명령어를 작성하시오

✅ 정답: `mkdir -p /var/www/html`

- `mkdir` 명령어는 디렉터리를 생성합니다[8](https://chunggaeguri.tistory.com/entry/%EB%A6%AC%EB%88%85%EC%8A%A4-%EB%AA%85%EB%A0%B9%EC%96%B4-mkdir)[14](https://velog.io/@jxxunnn/%EB%A6%AC%EB%88%85%EC%8A%A4-%EC%9E%90%EC%A3%BC-%EC%93%B0%EB%8A%94-%EC%9C%A0%EC%9A%A9%ED%95%9C-%EB%AA%85%EB%A0%B9%EC%96%B4-%EB%AA%A8%EC%9D%8C).
    
- `-p` 옵션은 필요한 경우 상위 디렉터리를 함께 생성합니다.
    
- 이 명령은 `/var`, `/var/www`, `/var/www/html` 디렉터리가 없을 경우 순차적으로 모두 생성합니다.
    

**📝 예시:**

text

`[user1@localhost ~]$ mkdir -p temp/mid/han [user1@localhost ~]$ ls -R temp temp: mid temp/mid: han temp/mid/han:`

**❗주의:** `-p` 옵션 없이 중간 경로의 디렉터리가 존재하지 않으면 오류가 발생합니다:

text

``[user1@localhost ~]$ mkdir temp/mid/han mkdir: `temp/mid/han' 디렉터리를 만들 수 없습니다: 그런 파일이나 디렉터리가 없습니다``

## 심볼릭 링크 옵션

## 심볼릭 링크를 만들 때 사용하는 옵션은?

✅ 정답: `-s`

- 심볼릭 링크(윈도우의 바로가기와 유사)는 `ln` 명령어에 `-s` 옵션을 사용하여 생성합니다.
    
- 기본 문법: `ln -s 원본파일 링크파일`
    

**📝 예시:**

text

`[user1@localhost ch2]$ ln -s data1 data1.sl [user1@localhost ch2]$ ls -l data1.sl lrwxrwxrwx. 1 user1 user1 5 11월 2 02:44 data1.sl -> data1`

**💡 하드 링크와 심볼릭 링크의 차이:**

|특성|하드 링크|심볼릭 링크|
|---|---|---|
|명령어|`ln 원본 링크`|`ln -s 원본 링크`|
|파일 종류 표시|일반 파일과 동일|`l`(소문자 L)로 표시|
|inode 번호|원본과 동일|원본과 다름|
|원본 삭제 시|링크 사용 가능|링크 사용 불가|
|파일 목록 표시|일반 파일과 유사|`링크파일 -> 원본파일` 형태|

## 디렉터리 이동/복사

## 현재 커서 아래(tmp) 디렉토리와 아래 파일들을 모두 / 아래로 (이동/복사)하시오

✅ 정답:

- 이동: `mv /tmp /`
    
- 복사: `cp -r /tmp /`
    

## 이동하기

- `mv` 명령어를 사용하여 디렉터리와 그 내용을 다른 위치로 이동할 수 있습니다.
    
- 단, 이동할 권한이 있어야 합니다(루트 권한 필요).
    

**📝 예시:**

text

`sudo mv /tmp /`

## 복사하기

- `cp` 명령어에 `-r` 옵션을 사용하여 디렉터리와 그 내용을 복사할 수 있습니다.
    
- 디렉터리 복사 시 `-r` 옵션은 필수입니다.
    

**📝 예시:**

text

`sudo cp -r /tmp /`

**❗주의:**

- 일반 사용자 권한으로는 시스템 디렉터리(/)에 쓰기 권한이 없어 실행할 수 없습니다.
    
- 관리자 권한(sudo)이 필요합니다.
    
- `/tmp`와 같은 시스템 디렉터리를 이동하면 시스템에 문제가 발생할 수 있습니다.
    

## 절대경로와 상대경로

## 절대경로와 상대 경로에 대해 아는대로 설명하시오

## 절대경로 (Absolute Path)

- 루트 디렉터리(`/`)부터 시작하여 파일이나 디렉터리의 전체 경로를 표시합니다[9](https://www.linkedin.com/pulse/difference-between-absolute-path-relative-linux-waqas-muazam).
    
- 항상 `/`로 시작합니다.
    
- 현재 작업 디렉터리와 관계없이 언제나 동일한 위치를 가리킵니다.
    
- 경로의 모든 디렉터리를 명시적으로 표시합니다.
    

**📝 예시:**

- `/home/user1/documents/file.txt`
    
- `/etc/passwd`
    
- `/var/log/syslog`
    

## 상대경로 (Relative Path)

- 현재 작업 중인 디렉터리를 기준으로 파일이나 디렉터리의 위치를 표시합니다[9](https://www.linkedin.com/pulse/difference-between-absolute-path-relative-linux-waqas-muazam).
    
- `/`로 시작하지 않습니다.
    
- 현재 디렉터리가 바뀌면 같은 상대경로도 다른 위치를 가리킬 수 있습니다.
    
- 경로 지정 시 특수 기호를 사용할 수 있습니다:
    
    - `.` : 현재 디렉터리
        
    - `..` : 상위 디렉터리
        

**📝 예시:**

- 현재 디렉터리가 `/home/user1`일 때:
    
    - `documents/file.txt` → `/home/user1/documents/file.txt`를 가리킴
        
    - `../user2` → `/home/user2`를 가리킴
        
    - `./config` → `/home/user1/config`를 가리킴
        

## 절대경로와 상대경로 비교표

|특성|절대경로|상대경로|
|---|---|---|
|시작 문자|`/`로 시작|`/` 이외의 문자로 시작|
|기준점|루트 디렉터리|현재 작업 디렉터리|
|일관성|항상 동일한 위치 지정|현재 위치에 따라 다른 위치 지정|
|경로 길이|일반적으로 더 김|일반적으로 더 짧음|
|사용 예|시스템 파일, 스크립트 내 고정 경로|관련 파일 간 이동, 사용자 작업|

## 숨김 파일 확인 옵션

## 숨김 파일을 확인하려면 어떤 옵션을 사용해야 하는가?

✅ 정답: `-a`

- 리눅스에서 파일이나 디렉터리명이 `.`(마침표)로 시작하면 숨김 파일로 간주됩니다.
    
- 숨김 파일은 기본 `ls` 명령으로는 표시되지 않습니다.
    
- `ls -a` 명령을 사용하면 숨김 파일을 포함한 모든 파일이 표시됩니다[14](https://velog.io/@jxxunnn/%EB%A6%AC%EB%88%85%EC%8A%A4-%EC%9E%90%EC%A3%BC-%EC%93%B0%EB%8A%94-%EC%9C%A0%EC%9A%A9%ED%95%9C-%EB%AA%85%EB%A0%B9%EC%96%B4-%EB%AA%A8%EC%9D%8C)[16](https://blog.naver.com/luvwithcat/222005727963).
    
- `-A` 옵션은 `.`과 `..`을 제외한 모든 숨김 파일을 표시합니다.
    

**📝 예시:**

text

`[user1@localhost ~]$ ls -a .           .bash_logout    .cache         .esd_auth      공개      바탕화면     서식 ..          .bash_profile   .config        .local         다운로드   비디오       음악 .ICEauthority  .bashrc        .dbus         .mozilla       문서        사진`

**비교:**

- 일반 `ls` 명령
    

text

`[user1@localhost ~]$ ls 공개 다운로드 문서 바탕화면 비디오 사진 서식 음악`

**💡 팁:** 숨김 파일 중에서도 `.`(현재 디렉터리)와 `..`(상위 디렉터리)는 특수한 의미를 가집니다.

## 디렉터리 삭제 불가 상황

## 디렉터리를 삭제할 수 없는 경우는 어떤 상황인가?

✅ 정답: 디렉터리가 비어있지 않은 경우

- `rmdir` 명령어는 **빈 디렉터리만** 삭제할 수 있습니다.
    
- 디렉터리 내에 파일이나 서브 디렉터리가 있으면 삭제할 수 없습니다.
    
- 이런 경우 "디렉터리가 비어있지 않음" 오류 메시지가 표시됩니다.
    

**📝 예시:**

text

``[user1@localhost temp]$ rmdir temp3 rmdir: failed to remove `temp3': 디렉터리가 비어있지 않음 [user1@localhost temp]$ ls temp3 hosts services temp text1 text2``

**디렉터리 삭제 방법:**

1. **빈 디렉터리 삭제:** `rmdir 디렉터리명`
    
2. **비어있지 않은 디렉터리 삭제:** `rm -r 디렉터리명`
    
    - `-r` 옵션은 재귀적으로 디렉터리 내 모든 파일과 하위 디렉터리를 삭제합니다.
        
    - **⚠️ 주의:** 삭제된 데이터는 복구가 어려우므로 신중히 사용해야 합니다.
        

**추가 삭제 불가 상황:**

- 삭제 권한이 없는 경우
    
- 시스템 보호 디렉터리인 경우
    
- 현재 작업 중인 디렉터리를 삭제하려는 경우
    

## 홈 디렉터리 기호

## 리눅스에서 홈 디렉터리를 나타내는 기호는 무엇인가

✅ 정답: `~` (물결표, tilde)

- `~` 기호는 현재 로그인한 사용자의 홈 디렉터리를 나타냅니다[14](https://velog.io/@jxxunnn/%EB%A6%AC%EB%88%85%EC%8A%A4-%EC%9E%90%EC%A3%BC-%EC%93%B0%EB%8A%94-%EC%9C%A0%EC%9A%A9%ED%95%9C-%EB%AA%85%EB%A0%B9%EC%96%B4-%EB%AA%A8%EC%9D%8C)[16](https://blog.naver.com/luvwithcat/222005727963).
    
- 홈 디렉터리는 각 사용자에게 할당된 개인 디렉터리로, 사용자 계정 생성 시 자동으로 지정됩니다.
    
- 일반적으로 `/home/사용자명` 위치에 있습니다.
    

**📝 사용 예시:**

- `cd ~` : 자신의 홈 디렉터리로 이동
    
- `cd ~user2` : user2의 홈 디렉터리로 이동
    
- `ls ~` : 자신의 홈 디렉터리 내용 표시
    
- `ls ~/documents` : 홈 디렉터리 안의 documents 디렉터리 내용 표시
    

**홈 디렉터리로 이동하는 다양한 방법:**

1. `cd ~` : 물결표 사용
    
2. `cd` : 인자 없이 cd 명령어만 실행
    
3. `cd $HOME` : HOME 환경변수 사용
    
4. `cd /home/사용자명` : 절대 경로 사용
    

**💡 참고:** 루트 사용자(root)의 홈 디렉터리는 일반적으로 `/root`입니다.

## ls 명령어 옵션 기능

## ls(-a -l, -F) 명령은 무엇을 출력하는가?

✅ 정답:

## 1. `ls -a` (all)

- 숨김 파일을 포함한 **모든** 파일과 디렉터리를 표시합니다[14](https://velog.io/@jxxunnn/%EB%A6%AC%EB%88%85%EC%8A%A4-%EC%9E%90%EC%A3%BC-%EC%93%B0%EB%8A%94-%EC%9C%A0%EC%9A%A9%ED%95%9C-%EB%AA%85%EB%A0%B9%EC%96%B4-%EB%AA%A8%EC%9D%8C)[16](https://blog.naver.com/luvwithcat/222005727963).
    
- 리눅스에서 `.`(마침표)로 시작하는 파일과 디렉터리는 숨김 상태입니다.
    
- `.`(현재 디렉터리)와 `..`(상위 디렉터리)도 표시됩니다.
    

**📝 예시:**

text

`[user1@localhost ~]$ ls -a .           .bash_logout    .cache         .esd_auth      공개      바탕화면     서식 ..          .bash_profile   .config        .local         다운로드   비디오       음악 .ICEauthority  .bashrc       .dbus         .mozilla       문서       사진`

## 2. `ls -l` (long format)

- 파일과 디렉터리의 **상세 정보**를 표시합니다[10](https://knowledgesharingvillage.com/tag/understanding-the-ls-command/).
    
- 표시되는 정보:
    
    - 파일 유형 및 권한
        
    - 하드 링크 수
        
    - 소유자 이름
        
    - 그룹 이름
        
    - 파일 크기(바이트)
        
    - 최종 수정 시간
        
    - 파일/디렉터리 이름
        

**📝 예시:**

text

`[user1@localhost ~]$ ls -l 합계 0 drwxr-xr-x. 2 user1 user1 6 10월 27 21:11 공개 drwxr-xr-x. 2 user1 user1 6 10월 27 21:11 다운로드 drwxr-xr-x. 2 user1 user1 6 10월 27 21:11 문서 drwxr-xr-x. 2 user1 user1 6 10월 27 21:11 바탕화면 drwxr-xr-x. 2 user1 user1 6 10월 27 21:11 비디오 drwxr-xr-x. 2 user1 user1 6 10월 27 21:11 사진 drwxr-xr-x. 2 user1 user1 6 10월 27 21:11 서식 drwxr-xr-x. 2 user1 user1 6 10월 27 21:11 음악`

## 3. `ls -F` (classify)

- 파일 유형을 식별할 수 있는 **특수 문자**를 파일명 끝에 추가합니다[10](https://knowledgesharingvillage.com/tag/understanding-the-ls-command/).
    
- 특수 문자:
    
    - `/` : 디렉터리
        
    - `*` : 실행 파일
        
    - `@` : 심볼릭 링크
        
    - 표시 없음 : 일반 파일
        

**📝 예시:**

text

`[user1@localhost ~]$ ls -F 공개/   다운로드/   문서/   바탕화면/   비디오/   사진/   서식/   음악/`

**💡 팁:** 옵션들을 조합하여 사용할 수 있습니다. 예: `ls -al`, `ls -alF`

## 디렉터리 생성 명령어

## 디렉터리를 생성하는 명령어는?

✅ 정답: `mkdir` (make directory)

- `mkdir` 명령어는 새로운 디렉터리를 생성합니다[8](https://chunggaeguri.tistory.com/entry/%EB%A6%AC%EB%88%85%EC%8A%A4-%EB%AA%85%EB%A0%B9%EC%96%B4-mkdir)[14](https://velog.io/@jxxunnn/%EB%A6%AC%EB%88%85%EC%8A%A4-%EC%9E%90%EC%A3%BC-%EC%93%B0%EB%8A%94-%EC%9C%A0%EC%9A%A9%ED%95%9C-%EB%AA%85%EB%A0%B9%EC%96%B4-%EB%AA%A8%EC%9D%8C).
    
- 기본 형식: `mkdir [옵션] [디렉터리명]`
    

**주요 옵션:**

- `-p` : 필요한 경우 부모 디렉터리도 함께 생성
    
- `-m` : 생성할 디렉터리의 권한을 설정
    
- `-v` : 상세 정보 표시 (verbose)
    

**📝 사용 예시:**

1. **단일 디렉터리 생성:**
    
    text
    
    `[user1@localhost ~]$ mkdir temp [user1@localhost ~]$ ls temp 공개 다운로드 문서 바탕화면 비디오 사진 서식 음악`
    
2. **여러 디렉터리 동시 생성:**
    
    text
    
    `[user1@localhost ~]$ mkdir tmp1 tmp2 tmp3 [user1@localhost ~]$ ls temp tmp1 tmp2 tmp3 공개 다운로드 문서 바탕화면 비디오 사진 서식 음악`
    
3. **계층 구조 디렉터리 생성 (-p 옵션):**
    
    text
    
    `[user1@localhost ~]$ mkdir -p temp/mid/han [user1@localhost ~]$ ls -R temp temp: mid temp/mid: han temp/mid/han:`
    

**💡 팁:** 디렉터리 이름에는 공백이나 특수문자를 사용하지 않는 것이 좋습니다. 만약 사용한다면 따옴표로 묶어주세요.

## 현재 디렉터리 위치 확인

## 현재 디렉터리의 위치를 출력하는 명령어는?

✅ 정답: `pwd` (print working directory)

- `pwd` 명령어는 현재 작업 중인 디렉터리의 절대 경로를 출력합니다[11](https://www.ibm.com/docs/ssw_aix_72/p_commands/pwd.html)[14](https://velog.io/@jxxunnn/%EB%A6%AC%EB%88%85%EC%8A%A4-%EC%9E%90%EC%A3%BC-%EC%93%B0%EB%8A%94-%EC%9C%A0%EC%9A%A9%ED%95%9C-%EB%AA%85%EB%A0%B9%EC%96%B4-%EB%AA%A8%EC%9D%8C)[16](https://blog.naver.com/luvwithcat/222005727963).
    
- 형식: `pwd [옵션]`
    

**주요 옵션:**

- `-L` : 심볼릭 링크가 있는 경우 링크 자체의 경로 표시 (기본값)
    
- `-P` : 심볼릭 링크가 있는 경우 실제 물리적 경로 표시
    

**📝 예시:**

text

`[user1@localhost ~]$ pwd /home/user1`

**용도:**

- 복잡한 디렉터리 구조에서 현재 위치 확인
    
- 스크립트에서 현재 경로 파악
    
- 상대 경로 작성 시 기준점 확인
    
- 명령어 실행 전 위치 확인
    

**💡 팁:** 쉘 프롬프트에 현재 디렉터리가 표시되어 있어도, 전체 경로를 정확히 알고 싶을 때 `pwd`를 사용합니다.

## 파일 생성 및 시간 변경

## 파일을 새로 만들거나, 수정 시간 변경에 사용하는 명령어는?

✅ 정답: `touch`

- `touch` 명령어는 빈 파일을 생성하거나 기존 파일의 수정 시간을 업데이트합니다.
    
- 형식: `touch [옵션] [파일명]`
    

**주요 기능:**

1. **파일 생성:** 지정한 파일이 없으면 빈 파일을 생성
    
2. **시간 업데이트:** 지정한 파일이 있으면 접근 시간과 수정 시간을 현재 시간으로 업데이트
    

**주요 옵션:**

- `-a` : 접근 시간만 변경
    
- `-m` : 수정 시간만 변경
    
- `-t [[CC]YY]MMDDhhmm[.ss]` : 지정된 시간으로 변경
    

**📝 예시:**

1. **빈 파일 생성:**
    
    text
    
    `[user1@localhost ch2]$ touch test [user1@localhost ch2]$ ls -l test -rw-rw-r--. 1 user1 user1 0 11월 2 02:55 test`
    
2. **기존 파일의 수정 시간 변경:**
    
    text
    
    `[user1@localhost ch2]$ ls -l data1.cp -rw-r--r--. 1 user1 user1 158 11월 2 02:41 data1.cp [user1@localhost ch2]$ touch data1.cp [user1@localhost ch2]$ ls -l data1.cp -rw-r--r--. 1 user1 user1 158 11월 2 02:56 data1.cp`
    
3. **특정 시간으로 변경:**
    
    text
    
    `[user1@localhost ch2]$ touch -t 12311200 test [user1@localhost ch2]$ ls -l test -rw-rw-r--. 1 user1 user1 0 12월 31 2019 test`
    

**💡 사용 용도:**

- 빈 파일 빠르게 생성
    
- 로그 로테이션 테스트
    
- 파일 백업 전 시간 확인
    
- 배치 작업에서 파일 시간 조정

# 1주차
## 4차산업혁명의 개념과 발전 단계

## 산업혁명의 정의
- 기계 발명과 기술 혁신에 의해 야기된 산업상의 큰 변화와 이에 따른 사회, 경제적 변화
## 산업혁명의 발전 단계
1. **제1물결(농업혁명)**: 수렵 채집사회에서 농경사회로의 전환
2. **제2물결(제1차 산업혁명)**: 증기기관 기반의 기계화 혁명(대량생산)
3. **제3물결(제3차 산업혁명)**: 컴퓨터와 인터넷을 이용한 정보화 혁명
4. **제4물결(제4차 산업혁명)**: 정보통신기술 융합으로 이루어지는 차세대 산업혁명(2012~미래)
## 4차산업혁명의 등장
- 2016년 1월 세계경제포럼을 통해 클라우스 슈바프가 처음 "4차산업혁명" 언급
- 2016년 3월 구글 딥마인드의 AI 알파고가 이세돌 바둑기사와의 대국에서 4:1로 승리
## 4차산업혁명의 정의
- 3차 산업혁명을 기반으로 디지털, 바이오와 물리학 사이의 모든 경계를 허무는 융합 기술 혁명의 시대
- 모든 삶의 방식이 근본적으로 변화하는 시대
- 인공지능이 인간의 고유한 능력을 뛰어넘을 수 있는 시대

## 4차산업혁명 시대의 특징
## 초연결(Hyper-connectivity)
- 사람, 사물, 공간 등 모든 것이 상호 연결되는 특성
- 사물인터넷, 클라우드 등의 기술을 통해 구현
## 초지능(Super-intelligence)
- 인공지능, 빅데이터 등을 통한 지능화 특성
- 기계 학습, 딥러닝 등 고도화된 알고리즘으로 구현
## 3차산업혁명과 4차산업혁명의 차이점
- 3차산업혁명: 정보의 수동입력, 자동화 중심
- 4차산업혁명: 자동 데이터 수집 및 저장, 처리, 분석, 개인별 맞춤형 생산 촉진

## 4차산업혁명 시대의 핵심 기술(ICBM+AI)
## 사물인터넷(IoT, Internet of Things)
- **정의**: 사람, 사물, 공간 등 모든 것이 인터넷에 연결되어 정보가 생성, 수집, 공유, 활용되는 초연결망
- **특징**: 이전 인터넷은 컴퓨터만을 연결했다면, 사물인터넷은 사람, 사물, 공간 등 모든 것을 연결
- **적용 사례**: 스마트홈, 웨어러블 디바이스, 자율주행차 등

## 클라우드 컴퓨팅(Cloud Computing)
- **정의**: 서버, 스토리지, 소프트웨어 등 필요한 IT 자원을 탄력적으로 제공받아 사용하는 컴퓨팅 환경
- **특징**:
    - 필요한 만큼의 컴퓨터 하드웨어, 소프트웨어, 데이터를 언제 어디서나 사용할 수 있는 환경
    - 사용자가 IT 자원을 직접 소유하지 않고 필요한 만큼 인터넷을 통해 사용하고 비용을 지불
- **장점**: 서버 구축 비용 절감과 관리의 효율성 향상
- **적용 사례**: 실시간 음악, 영화, 게임 서비스

## 빅데이터(Big Data)
- **정의**: 대량의 데이터로부터 가치를 추출하고 결과를 분석하는 기술
- **특징**: 인공지능 기술은 빅데이터로부터 만들어짐
- **중요성**: 4차산업혁명 시대의 석유로 비유됨
- **적용 사례**: 개인 맞춤 의료, 소비자 행동 분석 등
## 모바일(Mobile)

- **정의**: 사용자가 시간과 장소에 얽매이지 않고 서비스를 제공받을 수 있게 하는 기술
- **범위**: 웨어러블 디바이스, 자율주행차 등 모든 이동 가능한 매체 포함
- **특징**:
    - 사물인터넷, 클라우드, 빅데이터는 모두 모바일을 매개체로 서비스가 이루어짐
    - 스마트폰을 통해 모든 것을 관리하고 조정
    - 스마트폰은 상상을 현실로 만드는 중심 매개체
## 인공지능(AI, Artificial Intelligence)

- **정의**: 인간처럼 학습하고 사고하여 문제를 해결하는 능력을 가진 컴퓨터 시스템
- **특징**: 생각하는 과정을 통해 문제를 해결할 수 있도록 고안된 시스템
- **적용 사례**: 치매/뇌질환 진단 AI 솔루션, 음성인식 인공지능 비서(기가지니) 등

## 기타 중요 기술
- **로봇(Robot)**: 스스로 작업하는 능력을 갖춘 기계
- **드론(Drone)**: 로봇 기술을 응용한 무인 비행 물체
- **가상현실(VR)**: 실제와 유사한 인공적인 환경을 제공하는 기술
- **증강현실(AR)**: 실제 환경에 가상의 정보를 합성하여 원래 존재하는 사물처럼 보이도록 하는 기술
- **5G-6G**: 초고속, 초저지연, 초연결 통신을 위한 기반 기술
- **볼류메트릭(Volumetric)**: 카메라 수십 대 또는 수백 대를 이용해 실제 대상을 캡처하여 재현하는 기술

# 2주차
## 4차산업혁명의 개념과 기술적 기반
## 4차산업혁명의 정의와 진화 단계
- 4차산업혁명은 디지털·물리·생물학적 기술의 융합으로 생산·소비·의사소통 방식의 근본적 변화를 일으키는 패러다임 전환을 의미합니다. 
- 이는 1차(기계화), 2차(대량생산), 3차(디지털화) 산업혁명을 거쳐 2016년 클라우스 슈바프가 공식화했으며, 인공지능(AlphaGo의 이세돌 격파)과 사물인터넷(IoT)의 실용화로 본격화되었습니다. 
- 핵심 특징은 **초연결성(Hyper-connectivity)과 초지능화(Super-intelligence)로, 모든 사물이 데이터를 교환하며 자율적 의사결정을 수행합니다.**
## 핵심 기술(ICBM+AI)의 상호작용
1. **사물인터넷(IoT)**: 센서와 네트워크를 탑재한 장치들의 실시간 데이터 수집·공유 시스템(예: 스마트 홈, 웨어러블 디바이스).
2. **클라우드 컴퓨팅**: 온디맨드 IT 자원 제공 모델로, 서버 구축 비용 절감과 확장성 제공(예: AWS, Azure).
3. **빅데이터**: 다량의 정형/비정형 데이터에서 패턴 추출하여 의사결정 지원(예: 맞춤형 의료, 소비자 분석).
4. **모바일**: 5G·6G 통신과 결합된 이동형 플랫폼으로 모든 서비스의 중심 매개체 역할.
5. **인공지능(AI)**: 딥러닝과 신경망을 활용한 자기학습 시스템(예: 헬스케어 진단, 자율주행).
## 3차 vs 4차 산업혁명의 차별성
3차 산업혁명이 정보의 **수동적 처리**에 기반했다면, 4차는 **실시간 데이터 자동 수집→분석→피드백** 체계로 전환되었습니다1. 예를 들어, 공장 자동화에서 스마트 팩토리로의 진화는 M2M(Machine-to-Machine) 통신과 AI 예측 정비로 설명됩니다.
## 컴퓨터 하드웨어의 구조와 동작 원리

## 중앙처리장치(CPU)의 역할과 발전
- CPU는 **산술논리연산장치(ALU)**, **제어장치**, **레지스터**로 구성되며, 클럭 속도(Hz), 코어 수(멀티코어), 캐시 메모리 크기가 성능을 좌우합니다. 
- 64비트 아키텍처는 32비트 대비 더 넓은 메모리 주소 공간(18.4EB)을 처리할 수 있어 고성능 컴퓨팅에 필수적입니다. 
- 최신 CPU는 열 관리와 전력 효율을 개선한 7nm 공정(인텔 i9)까지 발전했습니다.
## 기억장치 계층 구조
1. **주기억장치**:
    - **RAM**: 휘발성 메모리로 DDR4 기준 3200MHz 속도와 16GB 이상 용량이 표준화.
    - **ROM**: BIOS 저장을 위한 비휘발성 메모리로 UEFI 펌웨어가 Legacy BIOS를 대체.
2. **보조기억장치**:
    - **HDD**: 7200rpm의 기계식 저장장치로 TB급 대용량 제공.
    - **SSD**: 3D NAND 기술로 3500MB/s의 순차 읽기 속도 달성.
    - **NVMe**: PCIe 인터페이스를 활용한 초고속 저장 프로토콜.
## 입출력 장치의 기술 혁신
- **입력 장치**: LiDAR 센서(아이폰), 홍채 인식(삼성 갤럭시), 햅틱 피드백(PS5 컨트롤러) 등 생체·환경 인식 기술 발전.

# 3주차
## 1. 알고리즘의 개념과 중요성
## 알고리즘의 정의
- 알고리즘이란 주어진 문제를 해결하기 위한 방법과 절차를 기술한 것입니다
- 컴퓨팅 사고 단계 중 문제 해결의 핵심 요소입니다
- 요리법(Recipe)이나 매뉴얼(Manual)과 유사한 개념으로 이해할 수 있습니다
## 알고리즘 작성 조건
- **입력**: 알고리즘에 입력되는 자료가 없거나 1개 이상 존재해야 합니다
- **출력**: 알고리즘이 실행되면 결과값이 1개 이상 출력되어야 합니다
- **유한성**: 알고리즘은 반드시 종료되어야 합니다
- **명확성**: 알고리즘의 명령은 모호하지 않고 명확해야 합니다
- **수행 가능성**: 알고리즘의 명령은 수행 가능해야 합니다
## 알고리즘과 프로그래밍의 관계

- 알고리즘은 문제 해결 방법을 논리적으로 기술한 것입니다1
    
- 프로그래밍은 알고리즘을 구체화하여 프로그램(Software)을 만드는 과정입니다1
    
- 효율적인 알고리즘 선택이 중요하며, 정확성과 효율성(빠르고 적은 자원 활용)이 핵심입니다1
    

## 2. 알고리즘의 표현 방법

## 자연어 표현

- 일상 언어를 사용해 알고리즘을 표현하는 방법입니다1
    
- 장점: 습득과 사용이 쉬움1
    
- 단점: 언어의 모호성으로 인해 불명확한 특징이 있음1
    

## 순서도(Flow Chart)

- 약속된 도형과 선분을 사용하여 문제 해결 과정을 표현하는 방법입니다1
    
- 장점: 짧은 프로그램의 처리 과정을 한눈에 파악하기 용이함1
    
- 단점: 복잡한 프로그램은 순서도로 작성하기 어렵고 수정이 어려움1
    

## 순서도 주요 기호

- **단말 기호**: 순서도의 시작과 끝을 의미1
    
- **흐름선**: 각 기호를 연결하며, 순서도의 흐름을 나타냄1
    
- **처리 기호**: 계산 등 자료의 연산 또는 처리를 나타냄1
    
- **판단 기호**: 조건을 판단하여 '예' 또는 '아니오'로 이동1
    
- **입출력 기호**: 자료의 입력과 출력을 나타냄1
    

## 의사코드(Pseudo Code)

- 알고리즘을 프로그래밍 언어와 유사한 형태로 풀어 쓴 것입니다1
    
- 장점: 프로그래밍 언어 구조와 유사하나 특정 문법을 따르지 않아 이해가 쉬움1
    
- 보는 사람이 쉽게 이해할 수 있어 선호하는 알고리즘 작성법입니다1
    

## 3. 알고리즘의 구성 요소

## 제어 구조

## 순차 구조(Sequence)

- 명령어들이 순서에 따라 차례로 실행되는 구조입니다1
    
- 문제 해결을 위한 명령어의 순서(일의 순서)를 의미합니다1
    
- 일부 명령어는 순서가 반드시 지켜져야 하며, 일부는 순서가 변경되어도 결과에 영향을 미치지 않습니다1
    

## 반복 구조(Iteration)

- 정해진 조건에 도달할 때까지 동일한 명령어가 반복적으로 실행되는 구조입니다1
    
- 동일한 행동이 반복되는 패턴을 인식하고 묶어서 표현할 때 효율적입니다1
    
- 예: 커피 자판기에서 잔돈이 남아있는 동안 커피를 계속 뽑는 상황1
    

## 선택 구조(Selection)

- 조건에 맞는 명령어 하나를 선택하여 실행하는 구조입니다1
    
- 문제 해결 과정에서 여러 선택적 상황이 발생할 때 사용합니다1
    
- 조건에 따라 다른 행동을 취하게 됩니다1
    

## 변수(Variable)

- 알고리즘 수행 시 발생되는 값을 저장하는 메모리 공간에 붙인 이름입니다1
    
- 값을 최근 데이터 하나만 저장할 수 있으므로 변수 내 값은 항상 변경될 수 있습니다1
    

## 데이터 구조

- **배열(Array)**: 같은 종류의 여러 데이터를 연속적으로 저장하는 저장소1
    
- **스택(Stack)**: 한쪽 끝에서만 데이터 삽입, 삭제할 수 있는 구조(LIFO)1
    
- **큐(Queue)**: 양쪽 끝에서 데이터 삽입, 삭제할 수 있는 구조(FIFO)1
    
- **리스트(List)**: 배열과 유사하나 불연속적으로 데이터 저장 가능한 구조1
    
- **트리(Tree)**: 데이터가 계층적으로 구성된 나뭇가지 형태의 데이터 저장 구조1
    

## 연산자

- **산술 연산자**: 더하기(+), 빼기(-), 곱하기(*), 나누기(/) 등1
    
- **논리 연산자**: 그리고(AND), 또는(OR), 아니다(NOT) 등1
    
- 그 외에도 관계 연산자, 문자열 처리 연산자, 리스트 연산자 등 다양합니다1
    

## 4. 알고리즘과 문제 해결

## 정렬 알고리즘

- n개의 데이터를 사용자가 정한 기준에 따라 순서대로 재배치하는 알고리즘입니다1
    
- 대표적인 정렬 알고리즘으로는 버블 정렬이 있습니다1
    

## 버블 정렬

- 인접한 두 개 데이터를 비교하여 크기를 비교한 후 교환하는 방식입니다1
    
- 과정: 인접한 두 항목을 비교하여 순서대로 위치하지 않으면 교환하고, 이 과정을 리스트 내 마지막 항목까지 반복합니다1
    

## 탐색 알고리즘

## 순차 탐색(Sequential Search)

- 찾고자 하는 값을 대상으로 리스트 내 항목들을 하나씩 차례대로 비교하여 탐색합니다1
    
- 장점: 알고리즘이 단순하여 구현하기 쉬움1
    
- 단점: 탐색 시간이 오래 소요됨1
    
- 정렬되지 않은 상태의 데이터를 대상으로 탐색할 때 적합합니다1
    

## 이진 탐색(Binary Search)

- 정렬된 데이터 집합에 대해 범위를 절반씩 줄여가면서 원하는 값을 찾는 방법입니다1
    
- 장점: 탐색 시간이 순차 탐색보다 빠름(절반으로 줄어듦)1
    
- 단점: 알고리즘이 순차 탐색보다 복잡함1
    
- 리스트 내 데이터가 반드시 정렬되어 있어야 정확한 결과를 얻을 수 있습니다1
    

## 5. 프로그램과 프로그래밍 언어

## 프로그램(Program)

- 어떤 업무 수행이나 전개를 위한 계획 또는 정해진 순서나 절차입니다1
    
- 컴퓨터 프로그램은 컴퓨터가 수행할 작업을 순서대로 기술해 놓은 것입니다1
    

## 프로그래밍 언어(Programming Language)

- 프로그램을 만들 때 사용하는 언어, 컴퓨터에게 작업을 지시하기 위해 사용하는 언어입니다1
    
- 기계어, 어셈블리어, C, C++, JAVA, 파이썬, 스크래치 등 다양합니다1
    

## 저수준 언어

- 사람이 이해하기 어려운 기계어 또는 기계어와 가까운 언어(기계 친화적)입니다1
    
- 기계어와 어셈블리어가 여기에 속합니다1
    

## 고급 언어

- 사람이 이해하기 쉽도록 만든 언어(인간 친화적)입니다1
    
- 대부분의 프로그래밍 언어가 여기에 속합니다1
    

## 텍스트 기반 언어

- 각 프로그래밍 언어의 문법을 지켜야 하므로 입문자들이 배우기에 까다롭습니다1
    
- C, Python, JAVA 등이 대표적입니다1
    

## 블록형 언어

- 프로그래밍 문법이 스크립트라는 이름의 그림 조각으로 이루어져 있습니다1
    
- 퍼즐을 맞추듯, 레고 블록을 조립하듯 프로그래밍이 가능하여 입문자에게 적합합니다1
    
- 스크래치, 엔트리 등이 대표적입니다1
    

## 언어 번역 방식

## 컴파일러(Compiler)

- 소스코드를 기계어로 번역하여 실행 파일을 생성한 후 한꺼번에 실행합니다1
    
- C, 자바 등에서 사용됩니다1
    

## 인터프리터(Interpreter)

- 소스코드를 한 번에 한 행씩 번역하여 실행합니다1
    
- 파이썬, 자바스크립트 등에서 사용됩니다1
    

## 피지컬 컴퓨팅

- 디지털 기술 및 장치를 활용하여 정보를 입력받아 처리한 결과를 출력하는 것입니다1
    
- 대표적인 도구로는 아두이노, 인텔 갈릴레오, 레고위두, 리틀비츠 등이 있습니다1
    
- 틴커캐드 서킷과 같은 시뮬레이터를 활용할 수 있습니다1

- **알고리즘 표현 방법**으로는 자연어, 순서도, 의사코드가 있습니다1

- **주요 알고리즘**으로는 정렬 알고리즘(버블 정렬 등)과 탐색 알고리즘(순차 탐색, 이진 탐색)이 있습니다1
    
- **프로그래밍 언어**는 저수준 언어와 고급 언어로 구분되며, 언어 번역 방식으로는

- **교육용 프로그래밍 언어**로는 블록형 언어(스크래치 등)가 있으며, **피지컬 컴퓨팅** 도구로는 아두이노가 주로 활용됩니다
